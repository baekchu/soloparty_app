## 🔑 필드 설명

### 필수 필드
| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `id` | string | 고유 식별자 (날짜-요일-지역-번호) | `"2026-01-24-fri-gangnam-1"` |
| `title` | string | 파티 제목 (최대 100자) | `"금요일 강남 미팅파티"` |

### 선택 필드 - 기본 정보
| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `time` | string | 시작 시간 | `"19:30"` |
| `venue` | string | 장소명 | `"파티룸 스카이라운지"` |
| `location` | string | 간략 위치 | `"강남"` |
| `region` | string | 지역 | `"서울"` |
| `address` | string | 상세 주소 | `"서울시 강남구..."` |

### 선택 필드 - 상세 정보
| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|

| `link` | string | 신청 링크 (인스타/카톡/네이버) | `"https://..."` |

## 📅 ID 명명 규칙

데이터 관리를 쉽게 하기 위해 ID 형식을 통일합니다:

```
[날짜]-[요일]-[지역]-[번호]
```

예시:
- `2026-01-24-fri-gangnam-1` (2026년 1월 24일 금요일 강남 1번째 파티)
- `2026-01-25-sat-hongdae-1` (2026년 1월 25일 토요일 홍대 1번째 파티)
- `2026-01-25-sat-gangnam-2` (2026년 1월 25일 토요일 강남 2번째 파티)

### 요일 약어
- 월: `mon`
- 화: `tue`
- 수: `wed`
- 목: `thu`
- 금: `fri`
- 토: `sat`
- 일: `sun`

## 📝 주간 데이터 관리 템플릿

같은 주에 여러 파티가 있을 때 복사해서 사용하세요:

```json
{
  "2026-01-24": [
    // 금요일 파티들
  ],
  "2026-01-25": [
    // 토요일 파티들
  ],
  "2026-01-26": [
    // 일요일 파티들
  ]
}
```

## ⚠️ 주의사항

1. **JSON 형식 검증**: 저장 전 JSON 형식이 올바른지 확인하세요
   - 온라인 JSON 검증기: https://jsonlint.com/
   
2. **날짜 형식**: 반드시 `YYYY-MM-DD` 형식 사용 (예: `2026-01-24`)

3. **문자열 이스케이프**: 설명에 따옴표 사용 시 `\"` 로 이스케이프

4. **줄바꿈**: `detailDescription`에서 줄바꿈은 `\n` 사용

## 🔄 Gist 업데이트 방법

1. GitHub Gist 페이지 접속
2. `gistfile1.txt` 파일 수정
3. JSON 데이터 붙여넣기
4. "Update secret gist" 클릭

앱은 3분마다 자동으로 데이터를 갱신합니다.

| 날짜 | 요일 | 시간 | 장소 | 제목 | 남자정원 | 여자정원 | 가격 |
|------|------|------|------|------|----------|----------|------|

| 2026-01-25 | 토 | 15:00 | 강남 | 게더링파티 | 6 | 6 | 25000 |

### 2️⃣ VS Code 찾기/바꾸기 활용

Ctrl+H로 일괄 날짜 변경:
- 이번 주 → 다음 주 날짜 변경
- ID의 날짜 부분도 함께 변경

### 3️⃣ 최소 데이터 형식 (간단 버전)
